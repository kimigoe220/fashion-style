# Fashion Style / Size Recommendation Project

本專案聚焦於服飾商品資料整理與尺寸推薦功能，目標不是只做「商品頁爬蟲」，而是進一步把不同品牌的尺寸資訊、試穿資料、庫存狀態整合為可用的推薦邏輯，讓使用者輸入身高與體重後，能獲得較實用的尺寸建議。

---

## 專案重點

本專案主要包含兩個層次：

1. **服飾資料爬蟲 / 結構化整理**
   - 擷取商品頁資訊
   - 擷取尺寸表 / 尺寸清單
   - 擷取試穿報告或版型資料
   - 整理為可存入資料庫的結構化資料

2. **尺寸推薦功能**
   - 使用者輸入身高、體重
   - 依品牌既有資料與規則進行推薦
   - 搭配庫存尺寸過濾，避免推薦無法購買的尺寸


---

## 爬蟲的資料結構


- 商品代碼
- 商品名稱
- 商品分類
- 可選尺寸
- 尺寸表
- 試穿報告
- 商品網址

---

## 尺寸推薦做的事是什麼？

尺寸推薦不是單純把尺寸抓下來，而是進一步回答：

> 使用者以這個身高、體重，應該買哪一個尺寸？

這一層的工作包含：

- 讀取資料庫中的商品尺寸資料
- 根據不同品牌特性決定推薦邏輯
- 比對使用者身形與既有試穿資料
- 只輸出目前仍有庫存的尺寸
- 若沒有合理尺寸，回傳不推薦或無庫存原因


---


## 品牌尺寸推薦邏輯差異

本專案目前針對不同品牌，設計了不同的推薦方式，而不是用同一套規則硬套所有品牌。

### 1. Lativ：以真人試穿資料做相近身形推薦

Lativ 的推薦流程會先從 `lativ_recommend` 資料表讀取商品資料，欄位包含品牌、商品代碼、分類、商品名稱、可購買尺寸、`tryon_samples` 與原始 `data`。程式會把使用者身高體重與試穿資料中的 `height_cm`、`weight_kg` 做距離計算，取最近的樣本做加權投票，最後只在目前有庫存的尺寸中挑選推薦結果；若候選尺寸都沒有庫存，則不推薦。

**特色：**
- 依據真人試穿資料
- 使用相近身形比對
- 有做近鄰加權投票
- 有做庫存過濾
- 若找不到合適可購買尺寸，會回傳不推薦原因

### 2. Pazzo：以試穿報告資料做近鄰投票推薦

Pazzo 的邏輯會從 `pazzo_tryon_reports` 讀取多筆試穿資料，欄位包含商品代碼、身高、體重、推薦尺寸、可購買尺寸、分類與網址。之後同樣用身高體重距離排序，取最近鄰資料做加權投票。若資料中有可購買尺寸，會優先挑選仍在 `available_size_list` 裡的尺寸；如果沒有可用庫存集合，則退回投票分數最高的尺寸。

**特色：**
- 也是以真人試穿報告為主
- 同樣有近鄰投票概念
- 優先考慮庫存尺寸
- 若沒有可用庫存集合，仍可回傳最佳候選尺寸

### 3. Uniqlo：以規則法（身高 + BMI）推薦

Uniqlo 的推薦方式與前兩者不同，不是依賴真人試穿近鄰，而是用規則法。程式先定義固定尺寸序 `XXS` 到 `3XL`，再依身高推估基礎尺寸，接著用 BMI 決定尺寸位移量。算出目標尺寸後，再從商品可用尺寸中找最近可選尺寸，最後再檢查該尺寸是否在目前庫存 `current_sizes` 中；若沒有庫存，會回傳「推薦尺寸目前無庫存」。資料來源為 `uniqlo_product_fit_data`。

**特色：**
- 不靠試穿近鄰資料
- 用身高 + BMI 的規則邏輯
- 適合資料格式較穩定的品牌
- 有明確尺寸序與鄰近尺寸回補機制
- 最後仍會做庫存檢查

## 不同品牌方法的比較

| 品牌 | 資料來源 | 推薦方法 | 是否考慮庫存 | 特點 |
|---|---|---|---|---|
| Lativ | 試穿樣本 + 商品尺寸資料 | 相近身形距離 + 加權投票 | 是 | 重視試穿樣本與實際可買尺寸 |
| Pazzo | 試穿報告 | 相近身形距離 + 加權投票 | 是 | 多筆試穿報告整合後投票 |
| Uniqlo | 商品尺寸資料 | 身高 + BMI 規則法 | 是 | 不依賴真人樣本，邏輯較穩定 |
